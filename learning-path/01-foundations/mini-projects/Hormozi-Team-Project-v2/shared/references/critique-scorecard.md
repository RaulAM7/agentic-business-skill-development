# Critique Scorecard — Weighted Scoring Framework

Framework de puntuacion ponderada para la skill proposal-critique.

## Estructura de pesos

### Value Equation (60% del total)

| Criterion | Weight | Que evalua |
|-----------|--------|------------|
| Dream Outcome claro y especifico | 12% | El resultado deseado es concreto, temporal (Dia 90), medible y emocionalmente resonante? |
| Perceived Likelihood (mecanismo + plan + control) | 18% | El cliente puede visualizar COMO va a pasar? Hay un proceso nombrado y secuencial? |
| Time Delay reducido (time-to-value) | 12% | Hay un quick win en 7-14 dias? O el cliente espera semanas sin ver resultados? |
| Effort & Sacrifice reducido | 10% | Esta claro que tiene que hacer el cliente? Los inputs estan listados y acotados? |
| Value Stack / percepcion de deal | 8% | Hay pricing, anchoring, cuantificacion de valor? O es un documento de scope sin numeros? |

### Proposal That Converts (40% del total)

| Criterion | Weight | Que evalua |
|-----------|--------|------------|
| Deliverables especificos + criterios de aceptacion | 10% | Cada entregable tiene "done when..."? O son descripciones vagas? |
| Scope y limites | 8% | Hay seccion "que NO incluye"? Hay proceso de change request? |
| Proof / credibilidad | 8% | Hay case studies, testimonios, portfolio, credenciales? O solo argumenta por logica? |
| Risk reversal / safeguards | 8% | Hay garantia objetiva? Checkpoint intermedio? Remedio antes del final? |
| Buying friction (pricing + next steps) | 6% | Hay precios, opciones de pago, CTA, proximos pasos? O el documento no puede cerrar? |

## Calculo del total ponderado

```
Total = SUM(score_i * weight_i) para cada criterio
```

Donde score_i esta en escala 0-10 y weight_i es el porcentaje.

## Interpretacion

| Score total | Interpretacion |
|-------------|----------------|
| < 4.0 | Propuesta con problemas fundamentales. No puede cerrar en su estado actual. |
| 4.0 - 5.5 | Propuesta con base solida pero multiples gaps criticos. Necesita revision significativa. |
| 5.5 - 7.0 | Propuesta funcional con areas de mejora claras. Puede cerrar con fixes puntuales. |
| 7.0 - 8.5 | Propuesta fuerte. Solo necesita pulido fino. |
| > 8.5 | Propuesta excelente. Lista para enviar. |

## Fragilidades

Cualquier criterio con score < 6 se marca como **FRAGILIDAD** y requiere fix en el Improvement Playbook.
