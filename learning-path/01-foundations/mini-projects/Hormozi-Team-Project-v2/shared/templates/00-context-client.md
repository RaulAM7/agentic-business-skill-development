<!-- SOURCE: 00.- IA360_Raul_Lopez_Client Context.docx -->

## Document Purpose
Este template captura el contexto factual completo del cliente: su voz, problemas, requisitos, creencias y transformacion deseada. El Documento de Contexto es la base para el Customer Map y el diseno de oferta.

**SCOPE**: Este documento es ESTRICTAMENTE FACTUAL. Extrae y preserva la voz del cliente. NO interpreta con frameworks Hormozi — eso es trabajo del Customer Map (siguiente fase).

**Prerequisite**: Materiales brutos en `deals/[NombreDeal]/00-context/raw/`
**Output**: `deals/[NombreDeal]/00-context/context-document.md`
**Next step**: `/agent customer-mapping`

## Completion Checklist

Before marking this Context Document complete, verify:

- [ ] Section 1 includes minimum 3 direct client quotes per subsection

- [ ] Section 2 identifies minimum 3 distinct structural problems with root causes
- [ ] Section 3 categorizes requirements into MUST/SHOULD/COULD

- [ ] Section 5 articulates transformation (From -> To)

## Gap Report (escribir en scratch/)

Producir un archivo separado `context-gaps.md` en la carpeta scratch/ del deal con:

- **Gaps identificados**: Que informacion falta para completar el contexto
- **Fuentes consultadas**: Que materiales se leyeron
- **Conflictos encontrados**: Si hay info contradictoria entre fuentes, como se resolvio
- **Siguiente accion sugerida**: Que deberia hacer el usuario para completar los gaps
